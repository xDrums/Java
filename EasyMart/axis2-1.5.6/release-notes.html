Apache Axis2 1.5.6 Release Notes

About this version

1.5.6 is a maintenance release that contains fixes for the following issues:

  • AXIS2-4263: Stopping ListenerManager does not cleanup several Timer threads
  • AXIS2-5067: content-type header is not generated with SAAJ API
  • AXIS2-5071: Failure to populate all members of an xs:sequence in a SOAP header
  • AXIS2-5074: Engaging Module for a Service Group not possible
  • AXIS2-5076: Number of pages broken in Axis2-admin web app.
  • AXIS2-5078: Axis takes 100% CPU, infinite loop in the QueryStringParser
  • AXIS2-5079: 1.5.5 does not allow exploded services to be dropped in services directory
  • AXIS2-5086: destroy(ServiceContext context) method never called in request scope.
  • AXIS2-5093: Connection cannot be cleaned up after 404 response

This release also upgrades Axiom from 1.2.11 to 1.2.12.

If possible users should upgrade to Axis2 1.6.1. The Axis2 1.5.6 maintenance release is intended for use in situations where upgrading to 1.6.x is not possible immediately.

About Axis2

Apache Axis2 is a complete re-design and re-write of the widely used Apache Axis engine and is a more efficient, more scalable, more modular and more XML-oriented Web services framework. It is carefully designed to support the easy addition of plug-in "modules" that extend its functionality for features such as security and reliability.

Modules supporting WS-Security/Secure-Conversation (Apache Rampart), WS-Trust (Apache Rahas), WS-Reliable Messaging (Apache Sandesha) and WS-Eventing (Apache Savan) are available via the Apache Web Services project. Please see the individual subproject sites for further information.
